Triangle UI

A scalable, white-label UI system for
residential smart home wallpads.

Built to serve any architecture client without rebuilding from scratch. Designed from the ground up by a team of three, with every decision made collaboratively.

RoleDesign System Lead
CompanyHyundai HT
Platform10" WallPad
Timeline2025 – 2026
01Context

One product.
Unlimited clients.

The wallpad is Hyundai HT's smart home interface, available in a 10" form factor. It ships as a fixed product to Hyundai Construction and as a white-label B2B product to architecture firms integrating it into residential developments.

Design Constraint

Every client arrives with different brand colors, feature sets, and CI guidelines. There's no fixed end state. The system has to be ready for clients we haven't met yet.

02Design Language

Simpler, Better.

Working within the design center's existing visual direction, our team of three built a shared design language from scratch, arriving at every decision together. My role was to translate that into the system: minimal tokens, minimal components, nothing that felt out of place on the hardware it lives on.

3-Layer System

Triangle-UI creates visual hierarchy through layered surfaces. Each layer uses a progressively lighter tone. The topmost surface uses a glass-like effect, a functional choice since shadows don't read clearly on the smoked-glass display hardware.

Categorized Colors

Category colors are part of the wallpad's design heritage: green for communication, blue for control, orange for information. We preserved the hues for continuity and recalibrated values to meet WCAG, achieving a 3.5:1 contrast ratio against each background.

Iconography

Icons use the roofline of a house as a consistent visual motif across the set. Usage drives the style: solid icons for buttons and actions, line icons for information. Built from basic geometric shapes to prioritize recognition over expression.

03Token Architecture

One system,
Configurable by Tokens.

The token system runs in two layers. When a new client configuration is created, their brand colors are added to the primitive layer and flow through the semantic tokens automatically.

Color

60-step primitive palette across neutrals and brand hues. Semantic tokens map intent: surface, brand, text, state, overlay.

Typography

Scale from 12px to 80px with named semantic roles. Font family and weight configurable per client at the semantic level.

Spacing

4px base grid with 8 named steps. Resolution modes (10" vs 13") switch spacing values by context. Same token, different output.

Radius

Corner radius from sharp to pill. Blur tokens for overlay and panel surfaces. Both configurable per client CI guidelines.

Most new client configurations are a token exercise.
Tokens cover color, typography, spacing, radius, and blur.

04Component Logic

Built to be Reconfigured,
Not Rebuilt.

Every component is built from a master with all possible slots included. Each instance exposes only what its context requires; the rest is hidden via Figma's property system. Every instance stays connected to one source, and client configurations become a matter of composition, not reconstruction.

05Version Control

Every Change
Documented.

Every major update is captured as a named branch. Client projects are managed separately, but every client file carries a version reference, so it's always clear which Triangle-UI release it was built against.

All changes are documented in the version history page.

06Final Design

Widget-Based
Home Screen.

The main screen is built from freely customizable widgets. Control widgets use category colors; information widgets stay neutral, letting users distinguish type at a glance.

Adaptive Control

Tapping a widget expands it into a detailed control panel. Dimmed icons indicate off states; category colors activate when a device is on.

Immersive Calling

The call interface uses the full 10" display. Floating buttons keep the field of view open, while the 3-layer system surfaces high-priority actions without cluttering the frame.

Clear Information

Orange highlights draw attention to critical info at a glance.
List-based components keep the experience consistent across the full interface.

07Reflection

Translating a
Design Language.

Building a design system from scratch meant every structural decision, from token naming and component slots to branching conventions, needed a rationale the whole team could stand behind. Nothing shipped until we agreed on why.

The sharpest tension was iconography: deciding to split the set into solid icons for actions and line icons for information. It was a small call with a large surface area. Once that rule was set, every icon that followed had to fit it.
